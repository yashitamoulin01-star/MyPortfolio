Overview

FocusSense is a productivity analytics platform that uses computer vision to detect and quantify focus state during work sessions. The system tracks head pose (pitch, yaw, roll) and eye gaze direction in real-time to infer whether a user is looking at their screen, looking away, or exhibiting distraction patterns, all processed locally via OpenCV and MediaPipe on the user's own machine.

Session analytics are stored in a local SQLite database and served through a FastAPI backend. A Next.js dashboard presents focus scores, distraction event timelines, and longitudinal session comparisons. No frame, image, or video data is ever transmitted outside the local machine. Privacy is guaranteed by architecture, not policy.

The Problem

Existing productivity tracking tools fall into two categories: browser-based monitoring (invasive, tracks websites visited and time spent) or webcam-based tools that stream video to remote servers for analysis (unacceptable for anyone who cares about privacy). Neither approach gives users objective focus data they can trust and act on.

The specific challenge: computer vision-based focus tracking has typically required cloud inference because running CV models locally was assumed to be too slow. MediaPipe changed this assumption. It provides real-time landmark detection performant enough for per-frame analysis on CPU without latency issues.

Privacy-by-architecture: The privacy guarantee in FocusSense doesn't come from a privacy policy or server-side encryption. It comes from the fact that video frames never leave the local Python process. No network call exists to compromise. This is a structural guarantee, not a promised one.

Technical Deep Dive

Head Pose Estimation
MediaPipe + solvePnP

MediaPipe Face Mesh provides 468 3D facial landmark coordinates per frame at real-time speeds on CPU. A subset of these landmarks (the eye corners, nose tip, chin, and cheek landmarks) forms a well-defined 3D face model. Using OpenCV's solvePnP function with these landmarks and a generic 3D face reference model computes the rotation matrix that best fits the observed 2D landmark positions to the 3D model.

The rotation matrix is decomposed into Euler angles: pitch (nodding up/down), yaw (turning left/right), and roll (tilting). Focus state inference uses primarily yaw and pitch: a yaw angle exceeding ±20° or pitch exceeding ±25° indicates the user is looking away from the screen. These thresholds were calibrated empirically across different monitor distances and face sizes.

Eye Gaze and Openness Tracking
Iris Landmarks + EAR

MediaPipe provides iris landmark coordinates within each eye, allowing gaze direction to be estimated from the relative position of the iris center within the eye bounding box. Iris position skewed strongly left or right indicates off-screen gaze even when head pose is approximately forward-facing.

Eye Aspect Ratio (EAR), the ratio of eye height to eye width computed from the eye landmarks, detects eye openness. Sustained low EAR indicates closed eyes (sleep, disengagement) while normal EAR variations correspond to natural blinking. The EAR threshold for "closed" was calibrated to distinguish normal blinks (100-150ms) from sustained closure (fatigue indicator at 300ms+).

Focus State Classifier
Temporal Window + Rule Engine

Per-frame head pose and gaze readings are noisy, and any single frame classification produces high false positive rates (a brief glance at a phone isn't a distraction event). The focus state classifier uses a 7-second temporal window with a majority-vote rule: a distraction is only logged if more than 65% of frames within the window indicate off-screen gaze. This dramatically reduces false positives from natural micro-movements.

State transitions have asymmetric hysteresis: entering "distracted" state requires 4.5 seconds of sustained off-gaze, but returning to "focused" state requires only 2 seconds of on-gaze. This asymmetry reflects the real cost of counted distractions: better to be slightly lenient on re-entry than to count brief recoveries as false "focused" periods.

Explainable Focus Score
Composable Metric Design

The session Focus Score is a composite of three sub-metrics: attention duration ratio (percentage of session time in focused state), distraction frequency (normalized distraction events per hour, penalizing frequent brief distractions more than rare long ones), and average distraction depth (mean duration of distraction events, capturing whether the user is briefly glancing away vs. fully disengaging).

Each sub-metric is shown alongside the composite score on the dashboard with a plain-language explanation of what drove it low. "Your score is 62. Distraction frequency is the biggest drag (14 events/hour vs. your 7-day average of 9). Most distractions occurred between 2:30 and 3:15 PM." This explainability layer was the feature users responded to most positively. A number without a reason is just a grade.

SQLite Session Storage and Analytics
Local-First Time-Series

Session data is stored in a local SQLite database with three core tables: sessions (session_id, start_time, end_time, focus_score), events (session_id, event_type, timestamp, duration), and aggregates (session_id, attention_ratio, distraction_count, avg_distraction_depth). The events table provides raw time-series data for detailed per-session analysis.

Pandas is used for the analytics layer: computing rolling averages, identifying distraction time-of-day patterns, and computing week-over-week score trends. The FastAPI backend exposes these as typed endpoints consumed by the Next.js dashboard. SQLite's file-based storage means session history is durable, portable, and requires zero infrastructure. The database is just a file in the user's home directory.

Challenges Solved

Head pose accuracy across face sizes and monitor distances: threshold calibration for pitch/yaw angles accounts for varying user-to-monitor distances; solvePnP with scaled 3D reference model handles face size variation robustly.
Distinguishing "thinking with eyes closed" from "distracted": EAR duration thresholds (300ms for fatigue indicator vs. 150ms for normal blink) combined with concurrent head pose data avoid penalizing legitimate closed-eye thinking.
False positives from natural micro-movements: 7-second temporal window with 65% majority threshold filters out brief natural head movements and brief intentional glances that don't represent genuine distraction.
Explainability for user engagement: raw scores without causal explanations don't drive behavior change. Per-sub-metric breakdown with time-of-day pattern analysis gives users actionable information rather than just a verdict.
Privacy guarantee by architecture: no network path for video data exists in the codebase. FastAPI only serves analytics from SQLite - never frames, never images. Privacy is enforced by code structure, not server configuration.

Lessons Learned

01
MediaPipe's 468-point face mesh is more than enough for robust head pose
Earlier approaches using only 68-point face models required careful feature selection. MediaPipe's 468-point mesh provides enough landmark density that solvePnP produces stable, accurate Euler angles even with partial face occlusion (glasses, hands near face).
02
Temporal context windows eliminate most false positives
Per-frame classification produced 3-4x too many distraction events. A 7-second majority-vote window reduced false positives dramatically with minimal impact on true detection latency - a natural glance away takes 1-2 seconds, a real distraction takes 5+ seconds.
03
Explainability is essential for productivity tools
Users disengaged when shown only a focus score without explanation. When the dashboard showed "most distractions between 2:30-3:15 PM" and "distraction frequency is 2x your weekly average," engagement and behavior change followed. The explanation was the product, not the number.
04
SQLite is the right database for local-first applications
Zero setup, zero maintenance, portable as a file, fast for time-series aggregations with proper indexing. For an application designed to run locally with no cloud dependency, SQLite's simplicity is a feature - there's nothing to configure, backup remotely, or maintain.

Future Roadmap

Phase 1
Personalized threshold calibration: a 3-minute calibration session at the start where the user looks at screen/away/reads establishes personalized head pose and gaze baselines rather than relying on population-average thresholds.
Phase 2
Pomodoro integration: focus session scheduling with automatic session segmentation and per-pomodoro score tracking, surfacing whether focus quality varies across work blocks.
Phase 3
Optional encrypted cloud sync: end-to-end encrypted sync of aggregated session stats (never frames) for multi-device history consolidation, giving users longitudinal trends across devices without any privacy compromise.
Phase 4
Team analytics dashboard: opt-in aggregated focus analytics for remote teams, showing team-level trends without exposing individual session data, useful for managers monitoring for burnout signals without surveillance.
